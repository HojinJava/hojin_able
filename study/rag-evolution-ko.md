# 한국어 RAG 발전 단계

> 같은 문서, 같은 질문 — 설정만 바꾸면 결과가 어떻게 달라지는가

---

## 실험 조건

**데이터:** 회사 계약서 50건 + 기술 보고서 30건 (전부 한국어)

**질문 3가지 (고정):**

```
Q1 (단순)  "납기 지연 패널티 기준이 뭔가요?"
Q2 (중간)  "A사와의 계약에서 우리가 불리한 조항들은?"
Q3 (복잡)  "전체 계약서에서 반복되는 법적 리스크 패턴은?"
```

## 각 레벨에서 무슨 일이 일어나는가

### LV1 — 기본 RAG
> 고정 크기 청크 + 영어 임베딩 모델

한국어의 문제: 같은 말을 다르게 써도 다른 개념으로 인식

```
"납기 지연 패널티"
"납기가 지연될 경우 위약금"   ← 같은 뜻인데
"지연 발생 시 배상금"         ← 영어 모델엔 완전 딴 말
```

---

### LV2 — 임베딩·검색 개선
> LaBSE + 형태소 분석 청킹 + 하이브리드 검색

**Q1·Q2가 크게 오르는 이유:**

```
변경 전: 고정 500자로 자르기 → 문장이 잘림
변경 후: 문장/단락 단위로 자르기 → 조항 원형 보존

변경 전: 영어 모델 → 한국어 표현 변형 인식 못함
변경 후: LaBSE → 같은 뜻이면 가깝게 인식
```

> LV1→LV2 점프가 한국어에서 유독 큰 이유:
> 영어는 같은 전환에 +20%p / **한국어는 +47%p**

---

### LV3 — 지식 그래프
> 개체·관계 추출 → Neo4j 저장 → 그래프 탐색

**Q2가 크게 오르는 이유:**

```
기존: "A사" 키워드로 텍스트 조각 검색
이후: A사 노드 → 연결된 계약서 → 조항 → 패널티 조건
      (5개 문서에 흩어진 내용을 한 번에 수집)
```

---

### LV3.5 — 온톨로지 자동화 *(단독 적용 가능)*
> 문서에서 스키마 자동 발견 → 멀티에이전트 정제

**수동 스키마 vs 자동 스키마:**

```
수동:  "지체상금" / "위약금" / "배상금" → 각각 다른 노드로 저장
자동:  "지체상금" / "위약금" / "배상금" → 손해배상조항(타입)으로 통일
```

같은 개념이 하나로 묶이면서 Q2·Q3 정밀도 향상.
LV3 없이 **LV2에 붙이는 것만으로도 효과**가 있어 독립 적용 가능.

---

### LV4 — 커뮤니티 감지
> Leiden 알고리즘으로 관련 문서 자동 클러스터링 → 클러스터별 요약

**Q3이 처음으로 제대로 작동하는 이유:**

```
50개 계약서를 주제별로 자동 분류:

  [납기·지연] 12건 → "납기 기준 불명확 8건, 지체상금 0.1~0.5% 분포"
  [손해배상]  18건 → "상한 없는 계약 15건, 면책조항 누락 7건"
  [기밀유지]   9건 → "기간 미명시 전부, 퇴직 후 의무 불일치"

Q3 질문 → 위 요약들 검색 → 전체 패턴 종합 답변
```

## 어디서 시작할까

```
내 상황                              → 추천
─────────────────────────────────────────────
한국어 문서가 있다                   → LV2 (무조건)
특정 회사·계약·인물 질문이 많다      → LV3
추출 품질이 들쭉날쭉하다             → LV3.5 (단독)
전체 트렌드·패턴이 궁금하다          → LV4
```
